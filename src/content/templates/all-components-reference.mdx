# MDXコンポーネント・記法リファレンス

このページは、ブログ記事作成時に利用できるすべてのMDXコンポーネントと記法をまとめたリファレンスです。記事執筆の際に参考にしてください。

## 基本的なマークダウン記法

### 見出し

```markdown
# 見出し1（ページタイトル用）
## 見出し2（主要セクション）
### 見出し3（サブセクション）
#### 見出し4（小見出し）
##### 見出し5（あまり使いません）
###### 見出し6（あまり使いません）
```

### テキスト装飾

```markdown
**太字**
*イタリック*
~~打ち消し線~~
`インラインコード`
```

実際の表示：
**太字**
*イタリック*
~~打ち消し線~~
`インラインコード`

### リスト

#### 番号なしリスト

```markdown
- 項目1
- 項目2
  - ネストした項目2.1
  - ネストした項目2.2
- 項目3
```

実際の表示：
- 項目1
- 項目2
  - ネストした項目2.1
  - ネストした項目2.2
- 項目3

#### 番号付きリスト

```markdown
1. 手順1
2. 手順2
   1. サブステップ2.1
   2. サブステップ2.2
3. 手順3
```

実際の表示：
1. 手順1
2. 手順2
   1. サブステップ2.1
   2. サブステップ2.2
3. 手順3

### リンク

```markdown
[テキストリンク](https://example.com)
[内部リンク](/blog/article-slug)
```

実際の表示：
[テキストリンク](https://example.com)
[内部リンク](/blog/article-slug)

### 画像（標準マークダウン）

```markdown
![代替テキスト](/path/to/image.jpg)
```

### 引用

```markdown
> これは引用文です。
> 複数行にまたがることもできます。
```

実際の表示：
> これは引用文です。
> 複数行にまたがることもできます。

### 水平線

```markdown
---
```

実際の表示：

## フロントマターの書き方

MDXファイルの先頭に以下のようなYAML形式のフロントマターを記述します。

```yaml
---
title: "記事のタイトル"
date: "YYYY-MM-DD"
tags: ["タグ1", "タグ2", "タグ3"]
thumbnail: "/images/thumbnail.jpg"
excerpt: "記事の概要。100〜150文字程度で記事の内容を簡潔に説明します。"
category: "カテゴリ名"
---
```

### 必須項目

- **title**: 記事のタイトル
- **date**: 公開日（YYYY-MM-DD形式）
- **tags**: 関連するタグの配列
- **category**: 記事のカテゴリ

### オプション項目

- **thumbnail**: サムネイル画像のパス
- **excerpt**: 記事の概要（指定しない場合は本文から自動抽出）

## テンプレートの使い方

当ブログでは以下の用途別テンプレートを用意しています：

1. [使い方ガイドテンプレート](/content/templates/howto-template.mdx) - 製品・サービスの使い方を解説する記事
2. [ニューステンプレート](/content/templates/news-template.mdx) - 最新ニュースを解説する記事
3. [レビューテンプレート](/content/templates/review-template.mdx) - 製品レビュー記事

新しい記事を作成する際は、目的に合ったテンプレートをコピーして使用してください。

## ベストプラクティス

1. **見出し構造を適切に使う**: H1はタイトル用に1つだけ使用し、コンテンツの構造はH2、H3、H4と順に使います
2. **画像には必ずalt属性を付ける**: 画像の内容を簡潔に説明する代替テキストを設定します
3. **長すぎる段落は避ける**: 読みやすさのために、3-4行ごとに段落を分けます
4. **強調はほどほどに**: 太字や斜体などの強調は重要なポイントのみに使用し、多用しないようにします
5. **リストを活用する**: 箇条書きが適切な場合は積極的に使用しましょう
6. **コードは適切にフォーマット**: コードはコードブロックで囲み、可能な場合は言語を指定します
7. **ノートは効果的に使う**: 重要な補足情報や警告には適切なノートコンポーネントを使用します

## 新しい記事の投稿手順

1. 適切なテンプレートを選択してコピーします
2. フロントマターを正確に記入します
3. コンテンツを作成し、適切なコンポーネントを使用します
4. 画像ファイルは適切なディレクトリに配置します
5. プレビューで表示を確認します
6. 校正して公開します
